# 4.2.17 - 账户状态提示界面

> 优先级: 🟡 中  
> 预估工期: 1 天  
> 依赖任务: 4.2.6  
> 状态: ✅ 已完成

## 任务概述

实现账户状态提示界面，根据不同状态（pending/disabled/locked/expired）显示相应的提示和操作。

## 目标

- [x] 待激活状态提示
- [x] 禁用状态提示
- [x] 锁定状态提示
- [x] 过期状态提示

## 技术要点

### 状态提示设计

```
🟡 待激活 (pending)
┌────────────────────────────────────────────────────────────┐
│  ⚠️ 账户尚未激活                                           │
│  您的账户需要完成激活才能登录。                              │
│  请查收激活邮件或联系管理员。                               │
│                                             [重新发送激活邮件] │
└────────────────────────────────────────────────────────────┘

⚪ 禁用 (disabled)
┌────────────────────────────────────────────────────────────┐
│  🚫 账户已被禁用                                           │
│  您的账户已被管理员禁用。                                    │
│  如有疑问，请联系系统管理员。                               │
└────────────────────────────────────────────────────────────┘

🔴 锁定 (locked)
┌────────────────────────────────────────────────────────────┐
│  🔒 账户已被锁定                                           │
│  由于多次密码输入错误，您的账户已被临时锁定。                 │
│  请在 25 分钟后重试，或联系管理员解锁。                      │
└────────────────────────────────────────────────────────────┘

🟣 已过期 (expired)
┌────────────────────────────────────────────────────────────┐
│  📅 账户已过期                                             │
│  您的账户已于 2026-01-01 过期。                             │
│  请联系管理员续期或申请重新激活。                           │
│                                                  [申请续期] │
└────────────────────────────────────────────────────────────┘
```

### 路由设计

```typescript
{ path: '/account-status', component: AccountStatusPage }
```

### 状态页面组件

```typescript
// 根据状态显示不同内容
<AccountStatusPage status="pending" />
<AccountStatusPage status="disabled" />
<AccountStatusPage status="locked" remainingMinutes={25} />
<AccountStatusPage status="expired" expiredAt="2026-01-01" />
```
